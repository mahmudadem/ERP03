# Architecture Documentation

This directory contains technical documentation for future developers who need to understand how and why things were built in ERP03.

## Structure

When creating architecture docs, include:

- **What changed**: Summary of the change
- **Why**: Rationale and context for the decision
- **File map**: Which files were affected and their roles
- **Migration notes**: If this change requires data migration or configuration updates
- **API contracts**: If this change affects the frontend-backend API

## Naming Convention

- `YYYY-MM-DD-feature-name.md` for feature-specific docs
- `CONVENTIONS.md` for project-wide coding conventions
- `DATA-MODELS.md` for data model documentation
- `API-CONTRACTS.md` for API contract documentation
