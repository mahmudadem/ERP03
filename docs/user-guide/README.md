# User Guide

This directory contains plain-language documentation for end users of the ERP03 system.

## Structure

When creating user guide docs, include:

- **Feature name**: What the feature is called
- **What it does**: Simple description of the feature
- **How to use it**: Step-by-step instructions
- **Who can access it**: Which roles can use this feature
- **Screenshots**: If applicable (add image files alongside the doc)

## Guidelines

- Write in plain language — avoid technical jargon
- Use step-by-step numbered instructions
- Include "Before you begin" prerequisites if needed
- Add "Tips" and "Common issues" sections when helpful
- Write for the person who uses the system, not the developer who built it

## Naming Convention

- `module-name.md` for module-specific guides (e.g., `inventory.md`, `sales.md`)
- `getting-started.md` for onboarding new users
- `super-admin.md` for Super Admin specific features
- `glossary.md` for term definitions
