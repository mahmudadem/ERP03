# Feature 01.5: Purchase Requisitions (Added from Gap Analysis)

## Overview
An internal request for goods or services before a formal Purchase Order is created. Enforces internal spending limits and approval workflows.

## Entities

### `PurchaseRequisition`
```typescript
{
  id: string; // auto-generated e.g., PR-0001
  companyId: string;
  departmentId?: string; // Optional internal department
  requestedBy: string; // UserId
  requestedDate: Date;
  requiredDate: Date;
  status: 'draft' | 'pending_approval' | 'approved' | 'rejected' | 'converted';
  lines: PurchaseRequisitionLine[];
  totalEstimatedAmount: number;
  currency: string;
  rejectionReason?: string;
  // Audit fields
}
```

### `PurchaseRequisitionLine`
```typescript
{
  id: string;
  itemId?: string; // Optional if requesting non-inventory item text
  description: string;
  quantity: number;
  uom: string;
  estimatedUnitPrice?: number;
  isConverted: boolean;
}
```

## Firestore Path
`companies/{companyId}/purchases/Data/purchase_requisitions`

## Services / Use Cases
- `CreatePurchaseRequisitionUseCase`
- `SubmitForApprovalUseCase` (Changes status to pending_approval)
- `ApproveRequisitionUseCase` (Changes to approved)
- `RejectRequisitionUseCase` (Changes to rejected with reason)
- `ConvertRequisitionToPOUseCase` 
  - Takes an approved PR and generates a Draft PO.
  - Links PO to PR (updates `PurchaseOrder.requisitionId`).
  - Marks PR as `converted`.

## API Routes
- `GET, POST /api/purchases/purchase-requisitions`
- `GET, PUT, DELETE /api/purchases/purchase-requisitions/:id`
- `POST /api/purchases/purchase-requisitions/:id/submit`
- `POST /api/purchases/purchase-requisitions/:id/approve`
- `POST /api/purchases/purchase-requisitions/:id/reject`
- `POST /api/purchases/purchase-requisitions/:id/convert-to-po`

## Frontend Pages
- **Purchases → Requisitions (List):** Show PRs, highlighting pending approvals.
- **Purchases → Requisition (Form):** Create/edit request.
- **Approvals Dashboard:** A view for managers to quickly approve/reject PRs.
