# Feature 01.5: Price Lists (Added from Gap Analysis)

## Overview
A system for managing customer-specific pricing, volume discounts, and promotional pricing across different currencies.

## Entities

### `PriceList`
```typescript
{
  id: string; // auto-generated
  companyId: string;
  name: string; // e.g., 'Retail', 'Wholesale', 'VIP'
  currency: string;
  isActive: boolean;
  validFrom?: Date;
  validTo?: Date;
  // Audit fields
}
```

### `PriceListItem`
```typescript
{
  id: string; // Compound: priceListId_itemId
  companyId: string;
  priceListId: string;
  itemId: string;
  uom: string; // e.g., applied to Box or Piece
  price: number;
  minQuantity: number; // For volume pricing (default: 1)
}
```

## Firestore Paths
- `companies/{companyId}/sales/Data/price_lists`
- `companies/{companyId}/sales/Data/price_list_items`

## Services / Use Cases
- `PriceCalculationService`
  - `getPrice(itemId, customerId, quantity, uom, currency, date): number`
  - Logic: 
    1. Look up customer's assigned active `PriceList`.
    2. Find matching `PriceListItem` for the item, uom, and minQuantity.
    3. If found, return price.
    4. If not found, fallback to Item's default `salesPrice` and convert via latest exchange rate if needed.

## API Routes
- `GET, POST /api/sales/price-lists`
- `GET, PUT, DELETE /api/sales/price-lists/:id`
- `GET, POST /api/sales/price-lists/:id/items`
- `PUT, DELETE /api/sales/price-lists/:id/items/:itemId`

## Frontend Pages
- **Sales → Price Lists:** Manage lists and their dates.
- **Sales → Price List Detail:** A data grid to manage prices for all items in the list. Features bulk import/update functionality.
